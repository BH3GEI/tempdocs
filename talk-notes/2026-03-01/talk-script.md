# MoFA Community Talk Script (Aligned with Announcement)

## 0) Opening (about 1 min)

```md
Hi everyone, thanks for joining today’s MoFA community talk.
We’ll follow the same 4-part agenda in the announcement:
1) Who we are and why we host this talk
2) GSoC idea updates
3) Next phase plans
4) PR review reminders, key rules, and Q&A
```

## 1️⃣ Who We Are & Why We’re Hosting This Talk (about 3-4 min)

```md
MoFA is an open-source framework for building AI agents.
Our current core direction is Rust-based mainline development, with Studio and local inference ecosystem evolving together.

Why this talk:
- align everyone on MoFA positioning and community values
- share roadmap updates transparently
- clarify this year’s GSoC rules early to reduce misunderstanding

Core value:
we prefer long-term builders who communicate clearly and iterate with the community.
```

## 2️⃣ Updates on GSoC Ideas (about 6-8 min)

```md
What changed:
- clearer scope boundaries (MVP / out-of-scope / acceptance criteria)
- stronger requirement for mainline integration and testable delivery
- expanded open tasks to include more production-facing framework work

Why we updated:
- reduce ambiguity in proposals
- help contributors choose feasible scope
- align contributions with real framework priorities

Status update:
- we have already updated the GSoC documents
- the updated links/docs have already been posted in the group

Proposal alignment reminder:
please include architecture blueprint, not feature list only.
```

## 3️⃣ Next Phase Plans (about 5-6 min)

```md
Next phase focus:
- framework-level architecture capabilities
- runtime integration quality
- observability, reliability, and maintainability
- community-driven exploration of meaningful killer apps

Beyond this GSoC:
if you are interested in long-term growth, you can continue through:
- issue triage and review collaboration
- docs/examples and contribution guide improvements
- feature iteration with repo collaborators and mentors

Collaboration note:
- we are also looking for long-term collaborators who want to co-maintain repositories with us
- all current maintainers/collaborators are part-time, so we need more stable help from the community
- we are recruiting volunteers partly because the current core maintainers are also part-time contributors
- this is a collaborator path, not a formal title track
- this is for post-GSoC collaboration
- it is NOT part of GSoC proposal selection/evaluation

Killer app call:
- in the next phase, we also hope community members propose meaningful killer app directions
- we care about real user value and strong product/architecture fit, not only feature stacking

We are building a sustainable community path, not a one-time event.
```
