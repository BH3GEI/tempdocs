# Volunteer Collaboration Mechanism (Discord Copy)

```md
🤝 [MoFA Volunteer Collaboration Group] Open Call (Pilot)

To improve review quality and response speed, we are opening a volunteer collaboration group.

Scope:
- issue triage and question routing
- PR pre-review and reproduction checks
- docs/examples/contribution guide improvements
- community Q&A support in Discord

Important rules:
- This is a **volunteer** role, not a maintainer title.
- All current maintainers/collaborators are part-time contributors with limited bandwidth.
- We are recruiting volunteers partly because the current core maintainers are also working part-time on this project.
- Join/leave is flexible (please notify in advance).
- Access is task/repo scoped and may be adjusted when needed.
- Merge behavior itself does **not** directly increase GSoC evaluation.
- GSoC evaluation remains based on quality, depth, communication, and consistency.

If interested, reply in this format:
- GitHub:
- Timezone:
- Repos you want to help:
- Weekly available hours:
- 1-2 representative PRs/issues:
- Why you want to help:

References:
- GSoC README: https://github.com/mofa-org/GSoC/blob/main/README.md
- Contributor docs (mdBook): https://mofa.ai/mofa/
```

## Short Chinese Version (Optional)

```md
【MoFA 志愿者协作组（试行）】

目标：提升 issue/PR 响应效率与评审质量。

范围：
- issue 分流与答疑
- PR 预审与复现
- 文档与示例改进
- Discord 社区协作

规则：
- 这是志愿者机制，不是 maintainer 头衔。
- 当前维护同学基本都是 part-time 协作，时间和带宽有限。
- 我们之所以征集志愿者，也是因为当前核心维护同学本身也是兼职投入这个项目。
- 可自由加入/退出（提前说明）。
- 权限按任务和仓库范围发放，可动态调整。
- merge 行为本身不直接计入 GSoC 加分。
- GSoC 仍按质量、深度、沟通、持续性评估。
```
